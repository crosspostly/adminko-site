# Развертывание проекта «Админ.Ко» (Контент-Фабрика)

Этот проект состоит из статического сайта и набора Node.js скриптов для автоматической генерации SEO/GEO контента с помощью ИИ.

## 1. Системные требования

Для работы скриптов-генераторов на сервере должно быть установлено:
- **ОС**: Linux (Ubuntu 20.04/22.04, Debian) или macOS.
- **Node.js**: Версия 18.0.0 или выше.
- **NPM**: Идет в комплекте с Node.js.
- **Git**: Для контроля версий.

Для самого сайта (HTML/CSS/JS) подойдет **абсолютно любой веб-сервер** (Nginx, Apache, GitHub Pages, Vercel, Beget).

## 2. Установка и запуск

### 2.1. Клонирование репозитория
```bash
git clone https://github.com/ВАШ_АККАУНТ/adminko-site.git
cd adminko-site
```

### 2.2. Установка зависимостей
Скриптам требуются библиотеки для запросов к API и работы с Gemini:
```bash
npm install
```
*(Установятся: `@google/generative-ai`, `axios`, и `tailwindcss` для сборки стилей).*

### 2.3. Настройка окружения
Скопируйте файл-пример и вставьте свои ключи:
```bash
cp .env.example .env
nano .env
```
Заполните:
- `GEMINI_API_KEY` — Ключ от Google Gemini.
- `YANDEX_API_TOKEN` — OAuth-токен от Яндекса (для Вордстата и Карт).
- `YANDEX_ORG_ID` — ID организации в Яндекс Бизнесе (по умолчанию 1071276806).

## 3. Сборка сайта (CSS)
Если вы вносили изменения в HTML (добавляли классы Tailwind), нужно пересобрать стили:
```bash
npm run build:css
```

## 4. Запуск Контент-Фабрики

### Генерация базы тем (Матрица запросов)
Создает тысячи вариантов на основе списка услуг:
```bash
node scripts/generate-massive-topics.js
```

### Мультиканальный оркестратор (Сайт, Дзен, Соцсети)
Берет темы и генерирует контент сразу для 3-х площадок:
```bash
node scripts/content-orchestrator.js
```
*Результаты сохраняются в папку `content_factory/`.*

### Авто-публикация новостей в Яндекс.Карты
Скрипт берет короткий пост и отправляет его в вашу карточку организации:
```bash
node scripts/yandex-maps-news.js
```

## 5. Автоматизация (Cron)
Чтобы скрипты работали сами по расписанию (например, писать по 5 статей каждый день в 03:00), добавьте задачу в cron на сервере:
```bash
crontab -e
```
Добавьте строку:
```bash
0 3 * * * cd /path/to/adminko-site && node scripts/auto-blogger.js >> logs.txt 2>&1
```

## 6. Деплой статики на сервер
Скопируйте содержимое папки `site/public/` в корневую директорию вашего веб-сервера (например, `/var/www/html/`).
```bash
cp -r site/public/* /var/www/admin-ko.ru/
```
